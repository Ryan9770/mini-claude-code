# 변경 이력 (history)

이 프로젝트(mini-claude-code)의 **수정사항·진행 상황을 날짜별로** 기록한다.

## 규칙
- **파일명**: `YYYY-MM-DD.md` — 하루 1개. 그날 처음 변경이 생길 때 만든다.
- 코드/스킬 수정, 커밋, 주요 결정, 측정 결과가 생기면 해당 날짜 파일에 항목을 **추가(append)**한다. (덮어쓰지 말 것)
- 하루가 끝났다고 정리하는 게 아니라, **변경이 발생할 때마다 즉시** 한 항목씩 쌓는다.

## 항목 형식
```
### HH:MM — <한 줄 제목>
- **변경**: 무엇을 바꿨나 (파일 경로 / 커밋 해시)
- **이유**: 왜 바꿨나
- **결과/검증**: 빌드·테스트·측정 결과 (있으면)
```

## 참고
- 소스 변경은 git 커밋과도 연결된다(커밋 해시 병기 권장).
- 측정·실험 결과나 인프라 설정 변경처럼 커밋에 안 남는 진행도 여기 기록한다.
